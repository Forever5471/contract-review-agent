# Review Reporting Skill

## 目标

把规则审查产生的风险项整理成审核结果。该 Skill 负责表达和汇总，不重新判断规则；当 GLM 已配置时，可对最终摘要做一次面向客户展示的润色。

## 输入

- `contract`：合同记录。
- `risks`：审查 Skill 输出的风险项。

## 输出

- `conclusion`：总体结论。
- `need_human_confirm`：是否需要人工确认。
- `risk_counts`：P0/P1/P2/P3 数量。
- `summary`：前端摘要。
- `llm_summary_used`：最终摘要是否经过 GLM 润色。
- `role_opinions`：业务、财务、法务、安全环保、归口/用印等角色意见。

## 内部工具

- `ReportBuildTool`：按优先级和角色归属聚合风险。
- `GLMChatClient`：仅对最终摘要进行表达润色，不参与风险判定。

## 约束

- 不新增规则风险。
- 不修改原始风险证据。
- P0/P1 风险保持人工确认标记，不自动放行。
- 大模型润色不得改变风险数量、风险等级、结论和人工确认状态。
- 大模型不可用时必须回退模板摘要，审核链路不能中断。
